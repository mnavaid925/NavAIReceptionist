{% comment %} Filter bar for the callback queue. The status select compares against `selected_status`, NOT `request.GET.status` like every other filter bar in the app. This page is the one place where an ABSENT parameter and an EMPTY one mean different things: no parameter at all is "I opened the queue" and the view resolves it to `pending`, while `?status=` is the All option and means no filter. `request.GET.status` is the empty string in both cases, so preselecting from it would leave Pending unmarked on the very view that is filtered to Pending. The view hands the resolved value over precisely so this template does not have to guess. All statuses is an explicit option rather than only a Reset link, because "show me the ones we already closed" is a normal thing to want and Reset takes you back to the pending queue, not to the full history. {% endcomment %}
