{% comment %} Filter bar for the bookings list. `status` is a string choice, compared with `==`. The provider/service/resource options are pks, so they compare with |stringformat:"d" — never |slugify. Every one of those querysets is passed from the view and re-authorised there before it filters, so a foreign pk degrades to no filter rather than leaking. The date inputs are LOCAL calendar days at this location. The view converts each to a half-open UTC range rather than using __date, which would convert in the wrong timezone and, on MySQL, return nothing at all. {% endcomment %}
