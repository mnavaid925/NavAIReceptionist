{% comment %} THE call-status badge. This partial is the single source of truth for the map — never inline the branches at a call site. in_progress -> badge-info completed -> badge-green abandoned -> badge-muted transferred -> badge-info failed -> badge-red Five statuses share four classes; badge-info is used twice on purpose. There is no badge-purple in this design system. The trailing else-branch fallback is mandatory even though all five values are branched: a value added to the model later must degrade to a readable pill rather than rendering nothing at all. Usage: {% include "partials/_call_status_badge.html" with obj=session %} {% endcomment %} {% if obj.status == 'in_progress' %} In Progress {% elif obj.status == 'completed' %} Completed {% elif obj.status == 'abandoned' %} Abandoned {% elif obj.status == 'transferred' %} Transferred {% elif obj.status == 'failed' %} Failed {% else %} {{ obj.get_status_display }} {% endif %}
