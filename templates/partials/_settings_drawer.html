{% comment %} Layout settings drawer. Every control is a plain button carrying data-option / data-value. static/js/layout.js delegates a single click listener over the whole document, writes the value to , persists it to localStorage and syncs aria-pressed across every control rendering the same option. Adding an option here needs no new JS — only a matching entry in the OPTIONS map in layout.js. Three control shapes, chosen by what the option actually is: - segmented ...... two or three text choices (on/off, light/dark) - preview-card ... arrangement choices, drawn as a CSS schematic so the difference is visible rather than a word to decode - swatch ......... colour choices, shown as the actual colour The pressed state is styled, not just announced, because a settings panel where nothing looks selected tells the user nothing about their current setup. {% endcomment %}
